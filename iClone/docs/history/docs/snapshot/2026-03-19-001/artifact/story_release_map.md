# ストーリーとリリース対応

## 主ストーリー

1. ユーザーがその場でメモを残す
2. save / sync の予定を理解したまま clone を続ける
3. PC / mobile の両方から同じデータを触る
4. 不要データを delete する
5. 次の質問と KPI 候補へ進む

## MRL 対応

| story cluster | MRL | delivered value |
|---|---|---|
| foundation | `MRL-0` | 独立 project、docs、schema、seed |
| connectivity | `MRL-1` | host stack、future connectivity route |
| edge UX | `MRL-2` | Android app shell、local persistence |
| PC UX | `MRL-3` | PC workspace UI、launcher |
| intelligence | `MRL-4` | question / KPI / transcript |
| closed loop | `MRL-5` | adb bridge、reverse sync |
| hardening | `MRL-6` | runtime file serving、validation、docs sync |
| sync-first UX | `MRL-7` | shared visual language、copy-first UI |
| clone-first UX | `MRL-8` | compact status、settings、delete、UX validation report |
