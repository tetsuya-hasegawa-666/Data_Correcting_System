# ストーリーとリリース対応表

## 目的

user story と release line の対応を俯瞰する正本とする。

## ストーリー層

1. 数秒でメモを残せる
2. テキスト、音声、写真を同一文脈に束ねられる
3. 近接時に自動同期される
4. PC 側で project 文脈に再配置される
5. AI が一問一葉で次問を返せる
6. KPI 候補と根拠ログが提示される

## MRL 対応

| story cluster | MRL | delivered value |
|---|---|---|
| foundation | `MRL-0` | north star、schema、seed data、ディレクトリ初期化 |
| connectivity | `MRL-1` | MAC whitelist と Syncthing contract |
| interaction | `MRL-2` | one-question UI、multimodal capture |
| intelligence | `MRL-3` | Whisper、次問生成、KPI candidate |
| hardening | `MRL-4` | security、retry、review evidence |
