# Develop History Snapshot

## Purpose

このディレクトリは、実開発変更の検証用として、変更した文書だけを保存する専用スナップショットレイヤである。

## Rules

- 対応する履歴は `develop/history/summary/summary.md` に集約する。
- 保存対象は変更した文書に限定する。
- ディレクトリ名は `YYYY-MM-DD-XXX` とする。
- `YYYY-MM-DD-XXX` は plan set の ID と一致してよい。

## Required Contents

- `manifest.md`
- 変更した文書のコピー
